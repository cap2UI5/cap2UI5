# tools/

The development harness for cap2UI5 — everything that is **not** deployed with
the [CAP project](../cap2UI5/). It mirrors the upstream
[abap2UI5](https://github.com/abap2UI5/abap2UI5) sources, transpiles the ABAP
classes to JavaScript, and copies the result into the `cap2UI5/` project.

For the full pipeline, npm scripts and GitHub Actions, see
[development.md](development.md). This file is just a map of the folders.

## Layout

| Path | What it is |
|---|---|
| [`run/`](run/) | **Working area — transient generated files.** Scratch space for the sync pipeline. Everything here is mirrored from upstream or produced by the transpiler and **rewritten fresh on every run** — treat it as build output, not as sources you edit by hand. |
| [`base/`](base/) | **The base skeleton.** A snapshot of the hand-maintained `cap2UI5/` project **with the generated content stripped out**, so the foundation everything is built on is visible in one place. Reference only — nothing reads from it. |
| [`scripts/`](scripts/) | The Node scripts behind the npm commands (mirror, transpile, copy, snapshot, …). |
| [`test/`](test/) | jest suite (backend units, sample apps, transpiler) + fixtures/helpers. |
| `jest.config.js` | jest configuration for the suite above. |

### `run/` — temporary files

`run/` is the pipeline's workbench. It has two halves:

- **`run/input/`** — snapshots mirrored from upstream: `abap2UI5/` (the framework
  `src/`), `app/` (the frontend `webapp`) and `samples/` (the whole cloud
  branch). Each folder is wiped and re-mirrored on every run.
- **`run/output/`** — the transpiled / prepared result (ABAP → JS, patched
  frontend). This is what gets copied into `cap2UI5/`.

Nothing in `run/` is meant to be hand-edited — the next pipeline run overwrites
it.

### `base/` — the base skeleton

`base/` shows the hand-maintained cap2UI5 **base project** — the skeleton the
sync pipeline fills the generated files into. The generated content is left out
so only the foundation remains:

- `app/z2ui5/webapp/` — omitted (fully generated by the frontend stream)
- `srv/app/samples/` — omitted (fully generated by the samples stream)
- `srv/z2ui5/` — **emptied** (the framework classes are generated by the
  backend stream; see the [placeholder](base/srv/z2ui5/README.md) there)

Refresh it with `npm run snapshot_base` whenever the base changes.

> [!NOTE]
> `base/` is a **reference snapshot, not a runnable project.** Because the
> generated framework classes under `srv/z2ui5/` are stripped, `cds watch`
> would fail there (`server.js` and `cat-service.js` require those classes, and
> the `package.json` exports resolve into `srv/z2ui5/`). To actually start the
> app, run the real project instead:
>
> ```bash
> cd cap2UI5
> npm install
> npx cds watch
> ```
>
> See the [cap2UI5 README](../cap2UI5/README.md) for the URLs it serves.
